# Implementation Plan: Settings Page with Strava Integration

**Branch**: `004-settings-strava` | **Date**: 2026-03-08 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/004-settings-strava/spec.md`

## Summary

Add a Settings route accessible via the header user dropdown. Two tabs: **User** (name, avatar, password change) and **Integrations** (Strava OAuth connect/disconnect + auto-sync of actual performance fields). Strava OAuth uses a Supabase Edge Function to handle the client-secret-protected token exchange server-side. Matching strategy: calendar day + activity type. The `strava_tokens` table gains a `user_id` FK; `profiles` gains `avatar_url`.

## Technical Context

**Language/Version**: TypeScript 5 (strict) + React 19
**Primary Dependencies**: React Router 7 (SPA), TanStack Query 5, Supabase JS 2, shadcn/ui (New York), i18next, Zod 4, date-fns 4
**Storage**: PostgreSQL via Supabase; Supabase Storage for avatars; Supabase Edge Functions for Strava OAuth
**Testing**: Mock mode (isMockMode auto-activates); pnpm typecheck
**Target Platform**: Web SPA (localhost:5173 dev; Vercel production)
**Project Type**: Web application (SPA, no SSR)
**Performance Goals**: Settings page loads < 1s; Strava sync < 5s per week; avatar upload < 3s
**Constraints**: Client secret must never reach the browser bundle; mock parity required for all new query files
**Scale/Scope**: Single user per settings page; Strava sync covers one week at a time; no background polling

## Phase 1: Design & Contracts

### Data Model
**Output**: [data-model.md](./data-model.md)

Key changes:

- `strava_tokens.user_id UUID REFERENCES profiles(id)` (with UNIQUE constraint — one Strava account per user)
- `strava_tokens.strava_athlete_name TEXT`, `connected_at TIMESTAMPTZ`, `last_synced_at TIMESTAMPTZ`
- New `StravaToken` TypeScript type in `app/types/strava.ts`
- `AuthUser.avatarUrl` added to auth types

### Interface Contracts
**Output**: [contracts/strava-edge-functions.md](./contracts/strava-edge-functions.md)

Three Edge Functions:
1. `strava-auth` — POST with `{ code, userId }` → exchanges code, stores tokens, returns `stravaAthleteName`
2. `strava-sync` — POST with `{ userId, weekStart }` → refreshes token if needed, fetches+matches activities, returns sync count
3. `strava-disconnect` — POST with `{ userId }` → deletes `strava_tokens` row (non-destructive: session data retained)

## Implementation Notes

### UserMenu → Dropdown
Current `UserMenu.tsx` is a flat div (no dropdown). Must be refactored to a shadcn `DropdownMenu` with:
- User name + role (display)
- "Settings" → navigate to `/settings`
- "Sign out" → existing logout

### Settings Route
- Registered in `app/routes.ts` — accessible to both coach and athlete (no role guard needed)
- Default tab: `user`; `tab=integrations` activates via URL param (to support OAuth callback)
- Tab state driven by URL `?tab=` param (not local state) so OAuth redirect works correctly

### Mock Mode Implementation
All new query files must include mock implementations:
- `profile.ts` mock: updates mock user in-memory; avatar upload returns a placeholder URL
- `strava-connection.ts` mock: simulates connected/disconnected state in sessionStorage
- Mock Strava sync: finds one unmatched session and populates it with sample data

### New Query Keys (add to `app/lib/queries/keys.ts`)
```typescript
profile: {
  all: ['profile'] as const,
  byId: (userId: string) => ['profile', userId] as const,
},
stravaConnection: {
  all: ['stravaConnection'] as const,
  byUser: (userId: string) => ['stravaConnection', userId] as const,
},
```

### New Training Types: `walk` and `hike`
Follows the standard pattern from CLAUDE.md §5:
1. Add `'walk'` and `'hike'` to `TRAINING_TYPES` const and `TypeSpecificData` union in `training.ts`
2. Add color config entries in `app/lib/utils/training-types.ts` (choose colors distinct from existing types)
3. Create `app/components/training/type-fields/WalkHikeFields.tsx` (shared component — both types have similar fields: terrain, elevation_gain_m)
4. Add translation keys to `en/training.json` and `pl/training.json`
5. Add cases in `SessionForm.tsx` switch for `'walk'` and `'hike'`

### i18n Namespaces
- Settings/Strava keys go in `common` namespace (app-wide, not role-specific)
- Walk/Hike type labels go in `training` namespace
- Translation keys: `settings.*`, `strava.*`, `training.types.walk`, `training.types.hike`
