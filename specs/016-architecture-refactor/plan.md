# Implementation Plan: Architecture Refactor — Module Structure

**Branch**: `refactor/architecture` | **Date**: 2026-04-04 | **Spec**: `specs/016-architecture-refactor/spec.md`

## Summary

Reorganise `app/` from a layer-first flat structure (`components/`, `lib/hooks/`, `lib/queries/`) into a `core/` + `modules/` topology. No logic changes — pure structural refactor. Every module owns its components, hooks, queries, tests, and i18n. Route files become thin re-exports. A typed `createQueryKeys` factory replaces ad-hoc `as const` arrays. The `analytics` module is renamed `stats` to avoid future confusion with instrumentation tooling.

**What changes**: directory layout, import paths, vitest config, test locations, query key factory.
**What does NOT change**: business logic, component APIs, hook behaviour, DB queries, rendering output.

**Structure Decision**: Module-first with role modules (athlete, coach) as page-composition layers over feature modules. Core holds all shared primitives. Route files are mechanical wiring only.

## Route Thin-Wrapper Pattern

All route files reduce to 1–3 lines. No logic permitted.

```tsx
// app/routes/athlete/week.$weekId.tsx
export { AthleteWeekPage as default } from '~/modules/athlete'

// app/routes/landing.tsx — route with meta
export { LandingPage as default, meta } from '~/modules/landing'
```

## createQueryKeys Factory

Custom implementation, zero external dependencies. Each module defines its own keys.

```ts
// API
const calendarKeys = createQueryKeys('calendar', {
  weeks: null,                                                    // static
  weekByDate: (id: string, start: string) => [id, start] as const, // dynamic
  sessions: (planId: string) => [planId] as const,
})

calendarKeys._root                           // ['calendar']
calendarKeys.weeks.queryKey                  // ['calendar', 'weeks']
calendarKeys.weeks._root                     // ['calendar', 'weeks']
calendarKeys.weekByDate._root                // ['calendar', 'weekByDate']
calendarKeys.weekByDate(id, s).queryKey      // ['calendar', 'weekByDate', id, s]
```
